JAVAWeb——会话

发表于 2020-04-13 Valine：

会话是什么？

一次会话包含多次请求和响应

* 浏览器第一次给服务器发送请求，会话建立，直到一方断开为止

会话的功能：

* 共享数据，在本地多次添加购物车后点击立即结算会把前面多次的点击结果共享。

方式：

* 客户端：Cookie
* 服务端：Session
Cookie

  1. 概念：客户端会话技术，将数据保存到客户端

    * 服务端1创建Cookie，绑定数据(new Cookie(String name,String value))
* 服务端1发送Cookie给客户端(resp.adCookie(Cookie cookie));原理response Header setCookie()
* 客户端收到Cookie后自动保存
* 服务端2收到客户端请求(Cookir[] request.getCookies())request Header Cookie()

  2. 细节

    1. 如何发送多个Cookie：多次创建并add。

    2. cookie保存多久：默认浏览器关闭即销毁

    3. 如何持久化

      • setMaxAge(int seconds)
      • 正数：将cookie数据存入硬盘，持久化存储n秒
      • 零：立即删除
      • 负数：默认

    4. 如何存储中文数据

      • Tomcat8支持中文
      • Tomcat8之前需要编码解码

    5. cookie的获取范围多大

      • 默认情况下cookie不会共享。
      • response中设置cookie.setPath为路径下所有项目共享。
      • response中设置cookie。setDomain为统一一级域名下所有项目共享。

  3. cookie的特点

    * cookie存储数据在客户端
* 对cookie大小有限制(4KB)
* 对同一个域名下的cookie数量有限制(20个)

  4. cookie的作用

    * 存储一些不太敏感的信息。
* 在不登陆的情况下，完成服务器对客户端身份识别。

Android——杂项

发表于 2020-04-12 Valine：

杂项1——动画

  1. Android有哪几种动画

    * View动画/补间动画：对View进行平移、缩放、选择和透明度的变化。
* 逐帧动画：按照一定顺序播放一组预先定义好的图片，类似于GIF
* 属性动画：对该类进行动画操作，真正改变了对象的属性。

    补间动画与属性动画的区别？

    * 补间动画：只产生了一个动画效果，其真实的坐标并没有发生改变（只是改变了View的显示效果而已，并不会真正的改变View的属性）。
* View做在做动画的时候，它并没有真正的移动它的位置，而是根据动画时间的插值，计算出一个Matrix，然后不停的invalidate。
* 在onDraw中的Canvas上使用这个计算出来的Matrix去draw这个View的内容。
* 但是onLayout中还是原来的位置，所以点击事件只能点击到原来的位置才能触发。

* 属性动画：通过反射技术来获取和执行属性的get，set方法真正的改变了对象位置的属性值。

    属性动画的插值器和估值器的作用。

    * 插值器：根据时间流逝的百分比计算出属性值改变的百分比。确定动画效果的变化模式——匀速变化，加速变化，补间动画和属性动画均可使用。
* 估值器：根据当前属性改变的百分比计算出改变后的属性值。
