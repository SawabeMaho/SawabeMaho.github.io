JAVA深入理解

发表于 2020-03-28 更新于 2020-03-29 分类于 JAVA Valine：

快速的跟着视频过了一遍java跟Android，很快就会忘记，现在系统梳理一下。

JAVA基础语法

标识符
  1. 标识符只能由A-Z,a-z,$ ,_,0-9组成且不能是数字开头
  2. 关键字不能做标识符
  3. 标识符区分大小写
    修饰符
    访问控制权限：public,protected,(default),private

整个世界都打包成一块，同时划分为四个层次: 联合国（public）【其他包】、国家（protected）【继承子孙】、洲际（default）【同一包】、个人（private）【当前类】。——摘自菜鸟教程笔记，

类型修饰：final(常量)，abstract(抽象)，static(静态)，synchonized（同步）
为了实现一些其他的功能，Java 也提供了许多非访问修饰符。
static 修饰符，用来修饰类方法和类变量。

final 修饰符，用来修饰类、方法和变量，final 修饰的类不能够被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，是不可修改的。

abstract 修饰符，用来创建抽象类和抽象方法。抽象方法 abstract void abm(); //抽象方法

synchronized 和 volatile 修饰符，主要用于线程的编程。
/**下面两个基本很少用****/
volatile 修饰符
volatile 修饰的成员变量在每次被线程访问时，都强制从共享内存中重新读取该成员变量的值。而且，当成员变量发生变化时，会强制线程将变化值回写到共享内存。这样在任何时刻，两个不同的线程总是看到某个成员变量的同一个值。

transient 修饰符
序列化的对象包含被 transient 修饰的实例变量时，java 虚拟机(JVM)跳过该特定的变量。

变量

java中主要有以下几个类型的变量
局部变量：

  1. 局部变量定义在方法中，直接{}包裹内代码块内也是局部变量
  2. 局部变量在方法或代码块执行时创建，执行结束后立刻销毁。
  3. 访问修饰符不能修饰局部变量
  4. 局部变量在栈上，例如main中 A a = new A();a就是一个局部变量，当main执行完毕立刻销毁。
  5. 局部变量没有默认值，必须初始化。

类变量(静态变量)：随着类的加载而存在，随着类的消失而消失。

  1. 类变量也叫静态变量，以static关键字声明，但必须在方法外
  2. 所有类实例共享同一份static变量
  3. 静态变量一般与final共用，其他很少使用。
  4. 第一次被访问时创建，在程序结束时销毁。
  5. 有默认值
  6. 可以通过classname.variablename 访问

成员变量:随着对象的创建而存在，随着对象的回收而释放。

  1. 成员变量声明在一个类中，但在方法，语句块之外。
  2. 当一个类被实例化后，成员变量的值就跟着确定。
  3. 成员变量可以声明在使用之后，编译器会把成员变量声明提前。
  4. 访问修饰符可以修饰实例变量
  5. 实例变量对于类内是可见的，如果为私有，必须设置set get方法。
  6. 实例变量具有默认值，数值为0，布尔为false，引用为null
    继承

一个类继承另一个类的属性和方法，其中继承者称为子类，被继承者为父类。

java面向对象概述

java在类中有三种变量,与C++不同的是JAVA的main()也需要写到类里边，成为类的公有方法且有一个固定的写法psvm，public static void main(String[] args)
三种变量：
局部变量：在方法中定义的变量，作用域在定义处开始，}处结束。
成员变量：在类内定义的变量，作用域是整个类内(会被同名局部变量覆盖)
类变量：加了static修饰符的成员变量，作用域是程序加载，注意局部变量不能加static修饰变成类变量。

构造方法

类中至少需要一个构造方法，如果没有显示写构造方法，那么编译器会提供一个默认无参构造。

源文件.java

一个源文件只能有一个public类
一个源文件可以有多个非public类
源文件的名称应该和public类的类名保持一致。
如果一个类定义在某个包中，需要先声明package packagename(一定要在首行)表面属于某个包。
import 放到package之后，类定义之前。

数据类型 四类八种

整型 byte：1字节，short：2字节，int：4字节，long：8字节
浮点型 float：4字节 double：8字节（浮点型用科学计数法表示，float能表示数字远大于long）
字符型 char 2字节，能够表示中文
布尔型 boolean 1字节

引用类型

除了上述四类八种外，java引入了引用类型，引用指向一个对象，指向对象的变量是引用变量，一旦一个变量指向一个对象，不能改变对象类型，不能Dog a，Cat a,
java完全放弃了C++中的指针。

常量

常量有final修饰或123 0x111 ‘A’这样的字面常量。

自动类型转换

byte，short，char->int->long->float->double
反向必须强转。

包装类

JAVA的基本数据类型四类八种没有对应的操作方法，JAVA为八种基本数据类型引入了包装类，包装类提供了一些方法能够共包装类对象调用。
不用特意去记，把包装类当作基本数据类型并且实现了一些方法就行了，编译器提供自动装箱拆箱。

面向对象

接口

接口时java中的一个特殊类型，java类继承只能单继承，而接口可以多继承。
类继承遵循is-a原则，Cat is a Animal，你不能让Cat在与Anima同级别的类上再次继承，说猫既是动物又是植物。C++里边是可以多继承的。

而接口继承遵循has-a原则，Cat has a method to Catch。
类用来继承属性和行为。
接口用来继承行为。

一般我们认为接口中只有抽象方法和常量，虽然后续引入了一些静态方法，默认方法之类的，但是用的最多的还是抽象方法。
接口不能实例化对象，没有构造方法，必须被继承，继承后的类称接口的实现类，可以通过实现类来实例化对象(抽象方法必须全部实现)，或者使用匿名内部类实现接口并实例化。
