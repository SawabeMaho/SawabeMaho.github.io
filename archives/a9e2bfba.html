C++面向对象编程

发表于 2020-03-07 更新于 2020-03-09 分类于 C++ Valine：

面向对象思想是C++与C最大的不同，所谓面向对象编程即使用抽象的方法来编程——从事物抽象出属性与行为。

面向对象的三大特性：

  • 封装 –将对象的属性(成员属性)，行为(成员方法)封装成类，管理权限
  • 继承 –子类继承父类的属性与行为，提高代码复用率,便于扩展
  • 多态 –通过父类的指针或引用调用子类的成员函数

类的访问权限

程序中的四区

  1. 代码区 存放代码
  2. 全局区 全局变量，常量，静态变量
  3. 栈区 局部变量，函数形参
  4. 堆区 new出的，需要手动释放delete

在C++中类与结构体的区别就是类的默认权限是private的而struct的默认权限是public的。

类的初始化

构造函数&&析构函数

构造函数：创建对象时为对象成员属性赋值

  1. 没有返回值
  2. 函数名与类名相同
  3. 可以有参数，可以重载
  4. 自动调用，且只调用一次

~析构函数：对象销毁前清理内存

  1. 没有返回值，不写void
  2. 函数名与~类名相同
  3. 不可以有参数，不可以重载
  4. 自动调用，且只调用一次
构造函数的分类

  1. 按参数：

    有参构造 Person(string int){}
    无参构造 Person(){}

  2. 按类型：

    普通构造 只要不是拷贝构造就是普通构造
    拷贝构造 Person(Person &p){}

构造函数的调用方式
  1. 括号法

    Person p1(10); √
    person p2; √
    Person p(); × 括号法不能使用无参构造，编译器分不清是不是一个函数声明例如int func();

  2. 显式调用

    Person p1=Person(10);
    Person p2;
    Person p2=p1;
    Person(10)//匿名对象>
    Person(p3) //这种用法是错误的，不能使用拷贝构造初始化匿名函数

  3. 隐式调用

    Person p4 = 10; 有参 等价Person p4 = Person(10);
    Person p5 = p4 拷贝 等价Person p5 = Person(p4);

什么时候会调用拷贝构造函数
  1. 使用一个对象初始化一个新对象
  2. 对象做函数值传递形式形参
  3. 对象做值传递形式函数返回值
    编译器是否提供默认构造函数
浅拷贝与深拷贝

浅拷贝：简单的赋值拷贝操作
深拷贝：在堆区重新申请空间，进行拷贝。
我们需要理解一下当我们谈到浅拷贝与深拷贝时默认类内有在堆区申请空间，否则深拷贝没有意义。

成员对象的概念

一个类A是另一个类B的成员，类A就是类B的成员对象。
成员对象的构造与析构顺序：
成员对象先构造，后析构，很明显的道理：类B的构造需要类A的参与，所以一定现有类A.

静态成员的概念

类中的属性或者方法前面有static关键字：

  1. 所有对象共享一份数据
  2. 在编译阶段分配内存 全局区
  3. 类内声明 static int m_A，类外定义int Person::m_A = 100;
友元的概念

友元即让其他类或成员访问本类私有成员或保护成员

运算符重载的概念

运算符重载是为了让运算符对自定义类型进行运算，例如自定义一个复数：
复数的加法是实部相加，副部相加。但是+本身并不能实现这个功能，这就需要我们重载+运算符。其中分为全局运算符重载与类内重载。

还有一点<<运算符只能使用全局重载
原因是成员重载cout<<p3;必须修改cout类。
如果不修改cout而是在p3类定义中修改
则等价于p3<<//p3做左操作数 成员重载成员做左操作数。

继承

子类不仅有父类的特性，同时又有自己的特性，便于扩展。
三种继承都无法访问父类私有成员，但私有也继承过来了占有内存
公有继承：公为公，保为保
私有继承：公保为私
保护继承：公保为保

因为子类含有父类，有儿子一定要现有父亲=>父类先构造
而栈中的空间是新进后出，所以儿子先析构
如果继承中出现同名属性/方法，需要加作用域区分，不加作用域默认是本类的成员属性/方法。

先继承的先构造

多继承问题

例如羊与驼继承子动物类，羊驼同时继承羊与驼，就发生了菱形继承，如动物有兽龄，那么羊驼会有两份兽龄占两份内存，如果要解决这个问题需要虚继承。

虚继承的目的是让某个类做出声明，承诺愿意共享它的基类。其中，这个被共享的基类就称为虚基类（Virtual Base Class），本例中的Animal 就是一个虚基类。在这种机制下，不论虚基类在继承体系中出现了多少次，在派生类中都只包含一份虚基类的成员，且值为最后修改的值。

多态

其实多态在类与对象之前就已经见过了

静态多态:

函数重载和运算符重载属于静态多态—-复用函数名
函数地址早绑定—编译阶段确定函数地址

动态多态：

派生类和虚函数实现运行时多态
函数地址晚绑定—运行阶段确定函数地址

我们可以提供父类的指针或引用调用子类，但如果子类重写了父类的函数，那么仍旧调用父类的函数，如果希望调用子类重写的函数，父类需要引入虚函数，所谓虚函数即函数声明前加virtual，
通过虚函数，我们可以使用基类的指针或引用来调用派生类的函数。

重载、重写、重定义的区别

重载：同一作用于下同名函数函数参数表不同造成重载。
重写：子类重写父类虚函数
重定义/隐藏：子类中的函数与父类函数同名，无论函数参数是否相同，父类中的函数不能在调用。
-函数名均相同-
重载：

  1. 作用域相同
  2. 函数参数表不同
  3. 不关心返回类型

重写：

  1. 作用域不同，一个在父类，一个在子类
  2. 除了{}内的不同，声明必须完全一致(子类不需要virtual声明)

隐藏：

  1. 如果子类与父类含有同名函数且父类函数不是虚函数，则子类无法通过son.Base::func()调用函数。
纯虚函数

如果我们只需要子类中的虚函数，不需要父类的虚函数，可以直接在父类定义中
virtual int func()=0;//纯虚函数
含有纯虚函数的类是抽象类，抽象类无法实例化对象，只能做基类，且抽象类的子类必须重写父类的纯虚函数，如果子类偏偏不重写，子类也是一个抽象类。

通过父类的指针调用子类还有一个隐患是如果子类中有开辟在堆区的内存，父类并不会调用子类的析构函数，如果不析构就会造成内存泄漏。
如果期望父类调用子类析构，父类的析构函数也要是虚函数。
父类析构前添加virtual关键字 就是 虚析构
如果父类的析构函数=0，就是纯虚析构，父类就成了抽象类。
