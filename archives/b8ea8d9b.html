JAVA——面试考点

发表于 2020-04-16 更新于 2020-04-17 Valine：
说一下JAVA-OOP三个特性
* 封装：权限管理，数据检验
* 继承：提高代码复用率
* 多态：本质(父类的引用调用子类对象)
String StringBuffer StringBuilder的方法
* String是字符串常量，在常量池中。适合不需要频繁修改的字符串
* StringBuffer和StringBuilder是字符串变量在堆区。
* StringBuffer是线程安全的，其内部加锁了。适合需要多线程缓冲
* StringBulider是非线程安全的。适合单线程缓冲
String a = “”;和String a = new (“”);的区别

String a = "";字符串存在于常量池，在元空间或永久代。
String a = new("");字符串存在于堆区。

* 常量池内只能存在不重复的字符串，同一个字符串的引用指向同一个地址。
* 使用==和equals比较结果如何？
* ==比较的是地址
* equals比较的是内容(String 重写了equals)
==和equals的区别
* == 基本类型：比较值，引用类型：比较地址。
* equals Object的方法之一，可以重写此方法实现引用类型的值比较。(为了舍弃C++中晦涩的运算符重载)
装箱拆箱什么意思？
* 是一个语法糖
* 编译器自动实现基本类型与包装类的转换
* 底层就是一个强制类型转换
Integer和int的区别
* Integer是int的包装类，在使用上基本没有啥不同
* 但是某些时候必须传递引用类型可以将int转为Integer
* Integer是引用类型默认为null，int是基本类型，默认为0
JAVA异常

异常分为错误，运行时异常，非运行时异常。
错误：程序无法恢复的异常，例如JVM错误，类定义错误，处理：停止运行。
运行时异常：程序自身bug造成，空指针异常，下标越界等。
非运行异常：可能会造成异常所以编译器建议添加trycatch的异常，FileNotFoundException等。

处理异常：
* 捕获异常:try…catch…finaly…
* 抛出异常:返回异常给调用者

JAVA反射
* 定义：在运行状态中，对于任意一个类都能直到它所有的属性和方法，对于任何一个对象都能够调用它的任何一个属性和方法。 
* 原理：JVM读取相应类的字节码文件动态的获得一个类，包括类的所有属性和方法，包括构造方法。
