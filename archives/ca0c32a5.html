Android——复习

发表于 2020-03-23 更新于 2020-03-29 分类于 Android Valine：

Andoid概述

Android是一个基于Linux的移动设备操作系统，主要用于移动设备,手机平板等。

Android结构

  1. 应用程序 app
  2. 应用框架 以java类的方式提供很多高级服务。
    • 活动管理者 控制应用程序生命周期和活动栈的所有方面。
    • 内容提供者 允许应用程序之间发布和分享数据。
    • 资源管理器 提供对非代码嵌入资源的访问，如字符串颜色设置和布局。
    • 通知管理器 允许应用程序显示对话框或者通知给用户。
    • 视图系统 一个可扩展的试图集合，用于创建应用程序和用户界面。
  3. 1 Android运行时 提供一个名为Dalvik的虚拟机，每个Android都会启动一个自己的虚拟机进程，互不影响。
  4. 2 Android程序库 是一个专门为Android开发的JAVA程序库。

    android.app - 提供应用程序模型的访问，是所有 Android 应用程序的基石。
    android.content - 方便应用程序之间，应用程序组件之间的内容访问，发布，消息传递。
    android.database - 用于访问内容提供者发布的数据，包含 SQLite 数据库管理类。
    android.opengl - OpenGL ES 3D 图片渲染 API 的 Java 接口。
    android.os - 提供应用程序访问标注操作系统服务的能力，包括消息，系统服务和进程间通信。
    android.text - 在设备显示上渲染和操作文本。
    android.view - 应用程序用户界面的基础构建块。
    android.widget - 丰富的预置用户界面组件集合，包括按钮，标签，列表，布局管理，单选按钮等。
    android.webkit - 一系列类的集合，允许为应用程序提供内建的 Web 浏览能力。

  5. Linux内核之上是一系列程序库的集合，包括开源的WEB浏览器引擎Webkit，仓库存储的SQLite数据库等。
  6. Android的最底层是一Linux内核，包括大约115个补丁的Linux。提供基本的系统功能，比如进程管理，内存管理，设备管理。

Android四大组件

Activities:描述UI，并且处理用户与机器屏幕的交互
Services：处理与应用程序关联的后台操作
Broadcast Receivers：处理Android操作系统和应用程序之间的通信
Content Providers：处理数据和数据库管理方面的问题
后续学习会着重学习，先了解一下。

Android 项目结构

  1. src: 包含项目的所有.java源文件，默认情况下，它包括一个MAinActivity.java源文件对应的活动类，当应用程序启动时从此处开始。
  2. gen:包含由编译器生成的.R文件，引用了所有的项目中的资源，该文件不能被修改。
  3. bin：这个文件夹包含Android由APT所构建的.apk包文件，以及运行ANdroid应用程序所需要的其他东西。
  4. res/drawable-hdpi:这个目录下包括所有的为高密度屏幕设计所需的drawable对象。
  5. res/layout:这个目录存放用于定义用户界面的文件。
  6. res/values:这个目录存放各种各样的包含一系列资源的XML文件，比如字符串和颜色的定义。
  7. AndroidManifest.xml:这个是应用程序的清单文件，描述了应用程序的基础特性，定义它的各种组件。

Activity

Activity 四种状态

只是我的理解，举例可能是错误的。

  1. 活动状态（Active/Running）主界面可以进行操作，打开淘宝，可以进行操作
  2. 暂停状态(Paused) 主界面失去焦点，突然淘宝弹出要求更新的对话框
  3. 停止状态(Stopped) 打开另一个程序淘宝进入后台
  4. 非活动状态（Dead） 关闭程序 上滑关闭淘宝

    Activity 七大方法

    onCreat():在活动第一次创建时调用。
    onStart()：在活动为用户可见时调用。
    onResume()：在应用程序与用户开始交互时调用。
    onPause()：当前活动要被暂停，上一个活动要被恢复时调用。
    onStop()：当活动不可见时调用。
    onDestory()：活动被销毁前调用。
    onRestart()：活动被停止以后重新打开时调用。

Service

Service概述

Service是一个后台运行的组件，执行长时间运行且不需要用户交互的任务，即使被销毁也依然可以工作。

Service两种状态

Started 安卓中的应用程序组件，如活动，提供startService()启动了服务，则服务是Started状态。一旦启动，服务可以在后台无线运行，即使启动它的组件已经被销毁。
Bound 当安卓的应用程序组件提供bindService()绑定了服务，则服务是Bound状态。

Service方法

onStartCommand():其他组件提供调用startService()请求启动服务时，系统调用该方法，如果你重写该方法，应该提供stopSelf()和stopService（）来停止服务。
onBind（）： 当其他组件想要通过bindService()来绑定服务时，系统调用该方法。如果你实现该方法，你需要返回IBinder对象来提供一个接口，以便客户来与服务通信。你必须实现该方法，如果你不允许绑定，则直接返回null。
onUnbind()：当客户中断所有服务发布的特殊接口时，系统调用该方法。
onRebind()：当新的客户端与服务连接，且此前它已经通过onUnbind(Intent)通知断开连接时，系统调用该方法。
onCreat()：当服务通过onStartCommand()和onBind()被第一次创建的时候，系统调用该方法。该调用要求执行一次性安装。
onDestory()：当服务不再有用或者被销毁时，系统调用该方法。你的服务需要实现该方法来清理任何资源，如线程，已注册的监听器，接收器等。
