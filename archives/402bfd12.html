Android面试总结

发表于 2020-04-06 更新于 2020-04-12 Valine：

Activity

  1. Activity有七种方法,四种状态 2. Activity A启动另一个Activity B会回调哪些方法？如果Activity B是完全透明呢？如果启动的是一个对话框Activity呢？ 3. onSaveInstanceState()有什么用？
    用来在应用由于意外操作——被回收，被意外终止时保存程序中的变量状态。
    在onStop()之前被调用，但与onPause()没有明确时序关系。
    onSaveInstanceState一般用于对临时变量的保存，比如EditText中输入的数据。
    onPause()用来做数据持久化，比如保存当前的游戏进度等。
    如何重建Activity？
    重写onRestoreInstanceState()方法来重新恢复Activity，
    或者直接在onCreate()中恢复State，需要注意的时，需要判断被保存的Bundle是否为空。一般推荐在onRestoreInstanceState()中进Activity重建。 4. Activity的四种启动模式 singletop:同一个栈内可以有多个实例，允许多次创建，该模式的Activity会默认进入启动它的任务栈，不会引起任务栈的变更，常用来防止多次快速点击启动过多activity。
    singletask：同一个栈里只能有一个实例，不允许重复创建，常用于主页和登录页面。
    栈顶复用及栈内复用会回调onNewIntent方法
    设置启动模式：
    FLAG_ACTIVITY_SINGLE_TOP:对应singleTop启动模式
    FLAG_ACTIVITY_NEW_TASK ：对应singleTask模式
    5. 如何启动其他Activity
    显式启动： 如果调用本应用的Activity，intent可以显式调用指定的.class对应的Activity
    隐式启动：通过指定intentFilter的action，category，data来确定一个Activity。

    Fragment

    Fragmet是Android中的一个重要组件，可以看作一款轻量级的Activity，例如微信底部的四个按钮，切换的是Fragment，而不是Activity。
    其生命周期依赖于Activity，所以比Activity更加复杂 1. Fragment与Activity的异同点
    同：都可以进行布局，显示，拥有生命周期，Fragment可以看作轻量级的Activity
    异：Fragment依附于Activity，其生命周期包含许多与Activity有关的方法，可以被Activity管理，因此fragment的方法都是public的。而Activity的方法都是protected的。
    2. Fragment的作用
    轻量级的界面展示，方便动态的加入移除交换。
    一个Activity可以同时出现多个Fragment，一个Fragment也可以被多个Activity调用。
    Activity的FragmentManager负责调用队列中的Fragment的生命周期方法，使其同步。
    3. 何时使用fragment？
    轻量的界面替换，例如从底部导航栏选择不同的界面。

Service

服务是Android四大组件之一
Service6个涉及生命周期的方法
onCreat()://首次创建服务时，系统调用此方法
onStartComand():当另一个组件调用startService()请求服务时，系统调用此方法。
onBind():当另一个组件调用bindService()绑定服务时，系统调用此方法。
onUnbind():当另一个组件调用unbindService()解绑服务时。
onRebind():服务被解绑后可以重新绑定
onDestory():当服务停止时调用
有两种启动方式：
startService（）:
onCreat()->onStartComand()->onDestory()
BindService():
onCreat()->onStartComand()->onBind()->onUnbind()->onDestory()

两种启动Service的方式有什么方式？
基本上没有什么不同，只不过bindService()可以在Activity中调用Service中的方法。
1、startService 启动的服务：主要用于启动一个服务执行后台任务，不进行通信。停止服务使用stopService；

2、bindService 启动的服务：该方法启动的服务可以进行通信。停止服务使用unbindService；

3、startService 同时也 bindService 启动的服务：停止服务应同时使用stepService与unbindService

如何通信：binder以及广播。

有哪些常见的系统Service

LAYOUT_INFLATE_SERVICE 获取布局
SENSOR_SERVICE 传感器
NOTIFICATION_SERVICE 状态栏服务

能否在SERVICE中进行耗时操作？
SERVICE默认在主线程中运行，如果进行耗时操作可能会导致ANR(主线程阻塞)。

AlarmManager的实现原理

通过调用AlarmManager的 set() 方法就可以设置一个定时任务，并提供三个参数（工作类型，定时任务触发的时间，PendingIntent对象）。其中第三个PendingIntent对象是关键，一般会调用它的 getBroadcast() 方法来获取一个能够执行广播的PendingIntent。这样当定时任务被触发的时候，广播接收器的onReceive()方法就可以得到执行。即通过服务和广播的循环触发实现定时服务。

前台服务是什么？和普通服务的不同，如何开启前台服务。

前台服务是那些被认为用户知道（用户所认可的）且在系统内存不足的时候不允许系统杀死的服务。
前台服务可以被用户觉察到，例如网易云音乐后台播放mp3可以在通知栏看到服务状态，
实现一个前台服务只需要在构建一个Notification后调用startForefround（）就会启动前台服务。

ActivityManagerService的作用。
AMS是Android最核心的服务，负责四大组件的启动切换，调度以及进程的管理和调度。

如何保证Service不被杀死，

  1. 提高优先级
  2. 将服务设为前台服务//对于通过startForeground启动的service，onDestory方法中需要通过stopForeground(true)来取消前台运行状态。
  3. 将Service写为系统服务
  4. 通过广播机制，每次被杀死都重新唤醒服务。

BroadcastReceiver

四大组件之三

广播类型：

  1. 普通广播：所有广播接收器几乎同时接收到广播。
  2. 有序广播：按照优先级从高优先级到低优先级传播，高优先级甚至可以终止该广播，虽然有一个指定接受对象的广播不能被终止，但是高优先级的广播接收者仍然可以将广播的data设为null，系统任务data为null的广播没有意义传不到指定接收者。
  3. 本地广播：发出的广播只能在本应用内部进行传递
  4. 粘性广播：广播滞留直到被接受
    广播注册的两种方式有什么区别
  5. 在代码中动态注册
  6. 在manifest中静态注册
    动态注册必须启动Activity才能接受广播，静态注册一旦安装就可以进行监听广播。

ContentProvider

四大组件之四
内容提供者：
主要负责存储与共享数据，与文件存储，sharedPreferences存储，SQLite数据库存储所不同的是ContentProvider主要用来暴露自己的内容给其他人，例如通讯录暴露联系人给QQ的寻找联系人。

数据持久化

  1. File I/O流 与JAVA保持一致
  2. SharedPreferences存储 轻量性数据存储，保存类型为XML键值对
  3. SQLite数据库存储 Android专用的轻量数据库
  4. ContentProvide 主要用来管理暴露自己的私有数据

SharedPreferences使用情形，使用中需要注意什么？

SharedPreferences是一种轻型的数据存储方式，适用于存储一些简单配置信息。
为什么SharedPreference在多线程下不可靠，Android对SharePreference有一个缓存策略，在内存中有一份该文件的缓存，所以在多线程下，读写不可靠,A insert一个数据，但B不能查到，应为B使用的是自己事先缓存好的。

SQLite

SQLite在做CUDA时是默认开启事务的，如果多条SQL语句要执行可以使用beginTransaction（）开启一个事务，并在结束后setTransactionSuccessful，此后如果结束事务或者commit会检查状态，如果成功则提交，否则会回滚。

SQLite如何删除一列

从需要删除的表中复制除需要删除的列之外的所有值，删除原表，新表重命名。

SQLite优化

  1. 批量操作使用事务转化为SQLiteStatemen
  2. 及时关闭Cursor，避免内存泄漏
  3. 耗时操作异步化
  4. 初始化容量做计算
